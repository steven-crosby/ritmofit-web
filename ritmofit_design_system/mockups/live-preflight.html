Hands-free after start.

RitmoFit owns the class timeline; provider SDKs own the audio. Before class, every track resolves to a named playable provider or a fixable verdict. During class, one rail chip always states what the music is doing — and a failure is a recoverable alert, never a dead end.

Player rail

The music always says what it’s doing.

♪SoundCloud ♪Preparing Apple Music… ♪Silence ♪Paused ♪Playback ended ♪Music off ⚠Playback error

One chip in the transport, Azeret Mono, no provider branding walls. “Silence” marks an authored free-timeline gap — a choice, not a mystery. “Music off” is prompter-only mode.

Runtime playback failure

Mid-class failure is serious — and recoverable.

⚠
Spotify stopped responding during “Pulso de Noche”. The class clock is still running. Pick a recovery — the prompter never stops.
Last resort for browser/session issues: open this track in Spotify. Handoff links live only inside this recovery surface — never as a primary action.

Unlike preflight verdicts, a runtime failure uses the danger channel: a role="alert" surface offering retry, reconnect, and continue-without-music.
